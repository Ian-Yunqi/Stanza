# 书方

> 情绪无人售药机 —— 不是书评网站，不是推荐引擎，是一台放在手机里的精神处方柜。

## 💡 愿景

嗨！

你是不是也有过这样的时刻：本来只是突然想找本书解解闷，结果光是在各种榜单里挑挑拣拣、对比评分和简介，就花掉了大半个钟头。

最后书没看成，人反而被"到底该看哪本"搞得更累了。找书的时间，不该比看书还长。

"书方"就是为了打破这种消耗而存在的。在这里，没有让人眼花缭乱的分类，也没有长篇大论的推荐语。你只需要凭借直觉，选一个你此刻的心情词，我就会立刻递给你一本对胃口的书。

帮你跳过所有纠结的步骤，一秒钟，直接掉进文字里。

---

## 📜 产品伦理

我们不希望困住你，所以"书方"坚持绝对的克制：

*   **不卖书**：这里没有购买链接，没有转化率指标。书只是药。药不管好不好吃，管用就行。
*   **不留你**：无账号、无推送、无社交。"用完即走"在这里不是一句口号，而是底层的产品伦理。
*   **不评价**：不对书的"好坏"做任何大众意义上的判断，只在乎它此刻是否契合你的心境。

---

## 🌌 核心体验

*   **呼吸镜面（首页）**
    八个情绪词以散落星空的形态，漂浮在全屏暖白色的镜面上。每个词有着不同的频率和相位，缓慢浮动——像一片安静的星系。每次打开或刷新，词的分布都会随机洗牌，为你呈现一张全新的星图。

*   **情绪处方（详情页）**
    一本书 = 一剂药。
    翻开处方，第一眼看不到书名和作者，只有一段"懂你"的导读。这是一种认知重塑，而非信息传播。当你顺着呼吸动画向下滑动，书名与 3-5 条金句才会伴随着你的阅读节奏，逐条安静地淡入。最后，一个黑色的方块 `□` 会温柔地提示你：这一段落结束了。

*   **今日一剂（兜底机制）**
    每天随机为你分配一本书，当天内保持固定。系统会默默记住你最近看过的 5 本书并自动避开，保证你总能遇见新的风景。

## 🛠 极简技术美学

为了匹配"用完即走"的理念，书方在技术上采用了极致的单文件架构（纯静态 H5）。

*   **大道至简**：整个产品只有一个 `书方.html`。零依赖、无构建工具，没有任何外部资源的完整性风险，甚至不需要网络也能在本地双击打开。
*   **流畅自然**：放弃臃肿的 JS 动画库，完全依赖 CSS `@keyframes` 实现星空漂浮的 GPU 加速动画，辅以 `IntersectionObserver` 精准控制金句的淡入节奏。
*   **向下兼容**：使用原生变量与函数，兼容所有移动端浏览器与老式 WebView。

## 📁 运转与 AI 协作机制

本项目的内容生产与代码迭代，深度融合了 AI 辅助工作流。

**文档驱动**
*   `CLAUDE.md`：AI 的全局系统指令与协作守则。
*   `需求文档.txt` / `UI与逻辑审查.md`：记录产品哲学、推演逻辑与交互状态机。
*   `技术选型与开发方案.md`：确立单文件架构与极简部署策略。

**数据沉淀**
*   书籍的导读与金句内容，由大语言模型（如 Gemini、DeepSeek 等）在构建期离线生成。
*   数据规范储存于 `书籍案例库.md`、`种子书库.md` 与 `样书案例模板.md`，保证最终交付给用户时，是零 API 调用的纯净体验。

## 🚀 部署与使用

由于是纯静态的单文件应用，部署极其自由：

1.  将 `书方.html` 托管至任意静态服务（如 GitHub Pages、Vercel、Cloudflare Pages 或阿里云 OSS）。
2.  无需繁杂的域名绑定，直接通过服务商提供的默认域名即可访问。
3.  在手机浏览器中打开，点击浏览器底部的"添加到主屏幕"，即可获得媲美原生 App 的沉浸式阅读体验。
