## 项目概述

你要开发的是一款名为"书方"的移动端 H5 网页产品。

**一句话定义：** 专治大学生"无聊、迷茫、厌倦"等情绪的精神处方。不是书评网站，不是推荐引擎，是一台"情绪无人售药机"。

**核心哲学：** 极简、高维认知重塑、无账号、无压力、用完即走。

## 产品核心规则（开发前必须理解）

> 以下规则决定"书方"为什么是"书方"而不是普通书评站。违反任何一条，产品灵魂就变了。

1. **标签点击 → 直接出导读（一本书），不展示列表** — 药剂师不给你一排药自己挑
2. **导读区不出现书名和作者** — 先用药，后知药名。书名在金句区才首次出现
3. **金句逐条淡入（IntersectionObserver）** — 3-5 条金句依次露出，不是一次性堆出来
4. **↓ 箭头固定在视口底部（position: fixed）** — 防止长导读把箭头挤出首屏
5. **视觉规范** — 暖白背景（#fafaf9）、衬线字体、页面最大宽 640px、纯文字向
6. **无账号、不黏住** — 仅用 localStorage 记录"最近 5 本已看"和"今日一剂"，不存任何用户数据

## 页面结构（一共 2 个页面）

```
首页 (/)               处方页 (/?tag=迷茫)
─────                  ─────────────────
"书方"                 认知重塑导读（200-400字）
                       无书名、无作者
8 个情绪标签             ↓ 箭头（固定底部，呼吸动画）
（无聊 迷茫 厌倦 浮躁   ─────────────────
 疲惫 空游无所依        第 1 条金句（淡入）
 烦闷 平淡）           第 2 条金句
                       第 3-5 条金句
每日一剂                （书名在此处首次出现）
（更小字号）            延伸信息（极小字）
                       □（终止符）
                       "换一剂 →" / "回到首页"
```

## 技术栈一句话

**纯静态 H5：1 个 HTML + 1 个 CSS + 1 个 JS + 1 个 books.json。** 无框架、无后端、无数据库。AI（Gemini 3.1 Pro）只在构建阶段生成导读内容，用户访问时零 API 调用。部署到 Cloudflare Pages。

## 特别提醒

- 书名和作者在**导读区不出现**，只在金句区开头首次出现。这是"书方"区别于书评网站的核心 UX 决策
- 金句逐条淡入用 `IntersectionObserver`，不是 `scroll` 事件
- ↓ 箭头是 `position: fixed`，不是文档流
- 情绪标签字体用无衬线 → 导读/金句用衬线。视觉区分"选择"和"阅读"两种模式
- 不装任何分析工具（GA、百度统计等），不设任何分享按钮。"用完即走"不是口号
