# 书方

> 情绪无人售药机 —— 不是书评网站，不是推荐引擎，是一台放在手机里的精神处方柜。

---

## 愿景

大学四年，你总有那么几个时刻——不知道该干什么、什么都不想做、心里空落落的。你滑开手机，短视频和信息流给你瞬间的麻醉，但放下之后，那种悬空的感觉还在。

**书方** 是一个完全不同的入口。你选一个情绪词（无聊、浮躁、疲惫……），它给你一本书。不是评分和简介，而是一位"愿意陪你聊几句的长辈"——一段写给你、比你还了解你此刻状态的文字，然后才是书名和金句。

- **不卖书**：没有购买链接，没有评分体系
- **不留你**：无账号、无推送、无社交。"用完即走"不是口号，是产品伦理
- **不评价**：不对书的"好坏"做任何判断。书只是药。药不管好不好吃，管用

## 核心功能

### 呼吸镜面（首页）

八个情绪词以散落星空的形态漂浮在全屏镜面上，每个词以不同的频率和相位缓慢浮动——像一片安静的星系。上下滑动或点击边缘切换词，点击中心进入对应处方。每次打开"全部"，词的分布随机洗牌，每次都是一张新星图。

### 情绪处方（详情页）

一本书 = 一剂药。页面呈现顺序经过仔细推敲：

1. **导读**（先用药）：不出现书名和作者，不出现任何评分或权威背书。你读到的第一段文字，是一个"比你还了解你此刻状态的人"在跟你说话。认知重塑，不是信息传播
2. **↓ 箭头**：固定在底部，呼吸动画提示"下面还有"
3. **金句区**（后知药名）：书名和作者在此首次出现。3-5 条金句逐条淡入，用 `IntersectionObserver` 控制节奏
4. **延伸信息**（极小字号）：在哪里可以读、如果你喜欢这本书可以试什么
5. **□ 终止符**：一个黑色的方块，告诉你"这一段结束了"

### 今日一剂

每天随机分配一本书，当天内保持固定。不重复展示最近 5 本已读内容。用 `localStorage` 实现，不设用户系统。

### 去重与降级

每个标签下可能有数十本书，系统优先匹配主标签、优先推荐未读过的书。如果主标签库已耗尽，自动降级到次标签推荐并给用户一个温和的提示。

## 技术架构

### 纯静态 H5

```
书方.html（单文件全部）
├── <style>    完整 CSS，含 mobile-first 响应式布局 + 纯 CSS 动画
├── <body>    2 个 view-container（首页 / 处方页），单页路由
└── <script>
    ├── BOOKS[]           40+ 本书的完整数据（导读、金句、元信息）
    ├── initMirror()      呼吸轮播 + 触摸交互 + 散落星空
    ├── matchBook()       标签匹配引擎（主标签 → 降级次标签 → 随机兜底）
    ├── renderHome()      首页渲染（随机初始化 + 随机回首页）
    ├── renderDose()      处方页渲染（导读 → 金句淡入 → 延伸信息）
    ├── initRouter()      URL hash → 视图切换（无框架 SPA）
    ├── applyRandomLayout()  多预设布局池（12 套坐标 + Fisher-Yates 洗牌）
    └── localStorage      seen books（最近 5 本）+ daily dose（7 天过期）
```

### 设计决策

| 决策 | 原因 |
|------|------|
| 单文件架构 | 零依赖、无构建工具、F12 即可调试 |
| `var` + `function` | 兼容所有移动端浏览器（含老式 WebView） |
| CSS `@keyframes` | 有机漂浮动画，GPU 加速，零 JS 开销 |
| `IntersectionObserver` | 金句淡入，不依赖 scroll 事件 |
| 无 CSP/无 SRI | 不加载任何外部资源，不存在完整性风险 |
| `#fafaf9` 暖白底 | 长时间阅读不刺眼，介于纯白和米黄之间的最佳阅读色 |

### AI 协作模式

- 书籍导读内容由 Gemini 在构建期生成，用户访问时零 API 调用
- 项目配置文档（`CLAUDE.md`、`需求文档.txt`、`UI与逻辑审查.md` 等）作为 AI 编码 Agent 的系统指令源
- 支持本地大模型辅助工作流：需求理解 → 方案输出 → 人类确认 → 代码生成

## 使用方式

1. 将 `书方.html` 部署到任意静态托管服务（Cloudflare Pages / Vercel / GitHub Pages / 阿里云 OSS）
2. 无需域名绑定即可通过服务商提供的 `.pages.dev` 或类似域名访问
3. 手机浏览器打开，添加到主屏幕即可获得接近原生应用的体验
